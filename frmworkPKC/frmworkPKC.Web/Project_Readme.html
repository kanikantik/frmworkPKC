1.Hyper

Hyper is a Visual Studio template created using Visual Studio which speeds up the development activities.

1.2 Highlights of the Release

The New Hyper Development Platform v1.0 has the following features:

  • Pre Added Repository, Service, Web and Tests Projects.
  • Option to customize the project preferences.
  • Sql,MySql and MongoDB(No SQL) are included in the project.
  • T4 templates for creating the Interfaces, Repositories and Services classes with basic code included.

  • Unity,Castle Windsor and NInject are included for Dependency Injection in the project.

  • New Relic

  • Swagger
  • Epam SSO
  • Sonar Analyzer

3.Detailed Information

3.1 Working with the Template

  • Please install all the items mentioned in the requirements section before you starting working on the template, so that there will be no errors once you start working with the template.
  • Close all the Visual Studio instances and double click the installer.

        
  • Once it is successfully installed, once visual studio and click on create new project.

  • In C# and .Net 4.5.2, you can see the Epam Hyper Web Application template, click that template and give the project name and location and click ok.

  • You will be navigated to customizations window, where you can select the Cloud Deployment, Unit test projects, UI Framework, Analytics and Underlying DB.

  • Once the customizations are completed, click on finish and it will create the projects that were explained in the next section.

  • Scaffolding through T4 templates:

         

         
  • There are T4 templates included in the application in Repositories, Services, Web and Service Unit test Projects.
  • Once the entity is created, by running the T4 templates the respective Repository, Services, Unit test and dependency injections will be created based on the entity name.
  • By default two entities are added in the Entities Project if SQL Server is selected as the underlying database and one entity is added if Mongo is selected.

         
  • Go to Repository Project, right click Repositories.tt file and click Run Custom Tool two folders will be created one for the interfaces and one for repositories with the files.
  • In the same way run the tool that are in Services, Web and Unit Tests files.

         
  • You can run all the templates once by going to Build -> Transform All Templates.
  • Once the templates are run, build the application to restore the nuget packages.
  • On success build, set the Web Project as the startup project (Right click on Web and Set as StartUp Project) and run the application.

  • Once the application opens in the browser and navigates to the default ASP.Net Page, you can see Glimpse at the bottom.

  • Navigate to http://localhost:67833/swagger , to navigate to swagger. Working on Swagger is described in this section.

3.3 Additional Features

3.3.1 Glimpse

  • This is a developer tool which helps to understand the server side analytics like the time taken by the method, what are the different views it is calling.
  • Glimpse will be on by default in the application.
  • You can make it off, by changing the RunTimePolicy in GlimpseSecurityPolicy.cs file in the Web Project.
  • For more information, visit website

3.3.2 MiniProfiler

  • MiniProfiler is a tool created and used by the StackExchange group of websites and is used for profiling your .NET and Ruby applications.
  • MiniProfiler.Integrations.MySql 1.0.1 reference is added to project if MySql and DatabaseSupport are selected
  • For more information visit http://miniprofiler.com/

3.3.3 Swagger

  • Swagger is used to represent the Rest API controller methods.
  • If you have node js installed in your machine, you can generate client side script through Swagger code, which will be explained below.
  • Before you go through the steps of Code Generation, check I you have node js installed in your machine.
  • Once node js is installed, below are the steps to generate code through Swagger.
  • Open command Prompt, navigate to the Web folder. If you use VS 2015, node_modules are downloaded by default else run the command ‘npm install’ in the command prompt.
  • After the command execution is completed successfully, run the command ‘npm install -g grunt-cli’.
  • Once grunt is installed, run the command ‘grunt default’. It creates a new file Model.js in lib folder.

3.3.4 Sonar Analyzer

  • This plugin enables the analysis of C# source files contained in .NET projects.
  • With the help of this plugin, a rule set is created which behaves as a code quality check for the programmer.
  • The rule set shows the critical and blocker violations as build errors so that the developer fixes them.

3.3.5 EPAM SSO

    Steps to work with EPAM SSO:
  • Hyper is registered with the following url with epam SSO - https://localhost:44310
  • Please go to Web Project -> Properties -> Web and change the project url to https://localhost:44310
  • Run the T4 templates and build the application.
  • Set Web Project as the start up project and run the applciation.
  • Once the home page is loaded, Click on the Menu Link EPAM SSO.
  • It logs in with the current logged in epam user and the user name can be viewed on the page.

3.3.6 Application Insights

  • Application Insights is an extensible Application Performance Management (APM) service for web developers
  • It works for apps on a wide variety of platforms including .NET, Node.js and J2EE, hosted on-premises or in the cloud.
  • For more information please visit https://azure.microsoft.com/en-in/services/application-insights/

3.3.7 New Relic
